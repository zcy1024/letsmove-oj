**您的题目需要满足以下规则：**

1. 通过严格的 $\mathit{gas}$ 来限制（如有需要）。

2. 题面 $\mathit{(.md)}$ 中除了题目要求之外，还应尽可能详细地描述输入/输出数据的格式，包括但不限于测试数据将通过`Main`还是`MainEvent`来对比答案。如果有可能，最好提供一组或几组样例数据。

3. 数据 $\mathit{(.zip)}$ 应包含所有测试数据，依次按照 $\text{1}.\mathit{in},\ \text{1}.\mathit{out},\ \text{2}.\mathit{in},\ \text{2}.\mathit{out},\ ...$ 命名。

4. $\mathit{.in:}$
   1. 每一行对应一个参数，每一行的第一个字符串表示类型，后续用空格隔开的数字/字符串表示数据，目前仅支持 $\mathit{u}\text{64},\ \mathit{string}$ 及其对应的 $\mathit{vector}$。
   2. `string Hello Sui` => 表示将通过`tx.pure.string("Hello Sui")`来传参。
   3. `u64_vector 1 3 5 7 9` => 表示将通过`tx.pure.vector("u64", [1, 3, 5, 7, 9])`来传参。
   4. 以此类推。
   5. **注意：** 文件最后不能有空行。

5. $\mathit{.out:}$
   1. 目前只支持一行一个数据来输出结果。
   2. 这一行分为三块内容：<br/>
      第一个字符串固定为`Main`或者`MainEvent`，表示从哪儿读取答案。<br/>
      第二个字符串表示其在`struct`中对应的`key`。<br/>
      剩下的内容表示其在`struct`中对应的`value`。
   3. 例如：`MainEvent say Hello Sui! Let's move to the moon!`<br/>
      它表示需要触发的事件为：
      ```move
      public struct MainEvent has copy, drop {
        say: String
      }
      ```
      并且其中`say`的值应为`Hello Sui! Let's move to the moon!`。
   4. `Main`同理。
